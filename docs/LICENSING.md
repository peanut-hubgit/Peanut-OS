# Política de licenciamento

## Código original

O código criado especificamente para o Peanut OS, incluindo as edições Compact, Touch Interface e Desktop, é licenciado sob **GPL-3.0-or-later**, salvo indicação explícita em contrário no próprio arquivo.

## Código contribuído

Ao enviar uma contribuição, o colaborador declara que possui o direito de compartilhar o conteúdo e concorda em licenciá-lo sob GPL-3.0-or-later.

## Componentes de terceiros

Bibliotecas, fontes, ícones, imagens, sons, temas e outros recursos de terceiros mantêm suas licenças originais. Eles devem ser registrados com:

- nome do componente;
- autor ou projeto de origem;
- versão utilizada;
- licença;
- endereço de origem;
- alterações realizadas.

Nenhum recurso de origem desconhecida deve ser incluído em uma build pública.

## Builds e forks

Distribuições compiladas, modificadas ou reempacotadas do Peanut OS devem disponibilizar o código-fonte correspondente e informar claramente que não são necessariamente builds oficiais.

## Marcas e identidade

A licença do código não concede automaticamente autorização para representar um fork como versão oficial do Peanut OS. Forks devem evitar confusão com o projeto original e identificar suas modificações.

## Releases

Uma release oficial deve incluir:

- código-fonte correspondente;
- instruções de instalação;
- lista de dependências;
- avisos de licença;
- identificação da edição;
- registro das principais alterações;
- indicação clara do estado: experimental, preview, beta ou estável.
